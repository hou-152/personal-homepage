WeChat Object Library / RAG Material Layer

把微信群信息流，变成可检索的对象库

这个 demo 不公开原始聊天记录。它展示的是本地管线如何把微信消息转成对象层： 每个对象保留来源锚点、事实等级、置信度和下一步动作，再进入 RAG 或长期知识库。

对象样例 JSON 评测摘要 返回主页资源

这不是日报

01 Raw

原始消息

本地 `raw.json` 是唯一事实源，不上传 GitHub，不作为公开展示材料。

02 Objects

对象层

案例、工具、观点、问题和行动项被拆成可复用对象，并绑定 local_id。

03 RAG

检索层

RAG 只索引对象层和入库版，不把 HTML 分享页当事实源。

04 Eval

评测层

用覆盖度、引用链、幻觉率和人工抽样表检查对象是否可靠。

RAG 应该吃什么

可入索引

objects.json

机器可读的标准化对象层。适合切 chunk、做 metadata filter、保留 citation anchor。

可入知识库

objects.md

人和 Agent 都容易读的入库版。适合 Logseq、Obsidian 或本地知识库。

不作事实源

fusion.html

分享页是展示产品，不是检索事实源。它可以被看，但不能替代原始证据链。

红线

raw.json

原始聊天记录只留本地，用于追溯和评测。公开版只能展示脱敏样例与质量摘要。
