# DESIGN：个人主页

## 1. 设计目标

这个网站要让访问者在很短时间内感受到三件事：

1. 这个人有能量。
2. 这个人成长速度很快。
3. 这个网站本身就是一个作品。

它不是传统简历页，不是博客，也不是生活记录站。它是一个面向家人、老师、朋友、老板的个人展示窗口，用来展示我通过 SpecDrivenCoding 做出来的个人主页，以及我个人的成长轨迹和价值潜力。

## 2. 整体气质

关键词：

- 酷
- 清晰
- 有速度感
- 有作品感
- 有成长线
- 稳定、可信、好打开

整体视觉不能太像模板站，也不能像普通求职简历。它应该更像一个个人成长展厅：第一眼有冲击力，继续往下看能看到证据和结构。

避免：

- 过度商务
- 过度学生气
- 过度可爱
- 过度炫技
- 大段自我感动文字
- 日记感、流水账感

## 3. 核心叙事线

页面的叙事顺序是：

1. 第一眼：这个网站有点酷。
2. 第二眼：这是一个正在快速成长的人。
3. 继续看：他不是嘴上说说，他真的做过很多尝试。
4. 看完后：这个人值得重新认识，也值得给机会。

页面不是按“个人简介、教育经历、技能、联系方式”的传统简历顺序来组织，而是按“感受建立 -> 价值证明 -> 行动入口”来组织。

核心定位：

个人品牌页为主，作品集作为证据。

## 4. 页面结构与设计说明

### 4.1 首屏 Hero

首屏目标：让人立刻知道这不是普通个人主页。

首屏内容包括：

- 一句强主标题
- 一句副标题
- 一个明确的视觉主元素
- 一个向下浏览的入口
- 一个说明“由 SpecDrivenCoding 做出”的轻量表达

主标题方向：

> 我正在把自己的成长轨迹，做成一个可以被看见的网站。

副标题方向：

> 这是一个用 SpecDrivenCoding 从 Brief、PRD、DESIGN 到开发实现的个人主页，也是我的作品集和个人展示窗口。

首屏要有完成度和冲击感，但不能影响加载速度。视觉可以使用动态线条、轨迹、卡片堆叠、成长路径图形等方式表达“上升斜率”。

### 4.2 我是谁

目标：快速建立个人定位。

内容不写成长篇自我介绍，而是用短句和标签表达。

结构建议：

- 我是谁
- 我现在关注什么
- 我正在训练什么能力
- 我希望别人如何理解我

表达方向：

> 我是一个正在用 AI、产品思维和持续实践，把想法变成作品的人。

这一屏要简短、有判断，不要写成个人作文。

### 4.3 成长线路

目标：展示高斜率。

这是网站的核心模块之一。

可以采用时间线、路径图、阶梯式卡片或成长地图。重点不是按时间罗列经历，而是展示变化：

- 起点
- 尝试
- 转折
- 项目
- 当前方向
- 下一步

每个节点包含：

- 时间或阶段
- 做了什么
- 学到了什么
- 体现了什么能力

这一部分要让人看到：我不是突然说自己有价值，而是一直在积累和变化。

### 4.4 项目展示

目标：给出证据。

项目展示使用项目卡片，而不是只放一串链接。

项目卡片的内容结构：

- 项目名称
- 一句话介绍
- 我做了什么
- 它证明了什么能力
- Demo 链接
- GitHub 链接

项目卡片可以理解为一个小展柜：

```text
┌──────────────────────────┐
│ 个人主页                  │
│ 用 SpecDrivenCoding 做出的展示网站 │
│                          │
│ 证明：表达能力 / AI 协作 / 开发推进 │
│                          │
│ [Demo]  [GitHub]          │
└──────────────────────────┘
```

GitHub 链接是可信证据，但不能抢页面主线。普通访问者先看项目价值，老师、老板或懂技术的人再点 GitHub 验证。

### 4.5 能力地图

目标：让老师和老板快速判断我有什么潜力。

不建议使用饼状图。能力不是比例，写成“产品能力 30%、表达能力 20%”会显得虚假。

更适合做成一组能力卡片：

```text
[结构化表达]
我能把一个模糊想法拆成 Brief、PRD 和设计文档。

[AI 协作]
我能用 AI 辅助完成从想法到网页的完整流程。

[项目推进]
我能把想法推进成可以打开、可以展示的作品。
```

能力方向包括：

- 结构化表达
- AI 协作
- 项目推进
- 信息架构
- 页面审美
- 公开展示
- 持续迭代

这一屏要体现的是“我能把能力组织成结果”，而不是单纯罗列工具。

### 4.6 成长热力图 / Build Log

目标：证明我在持续推进。

用类似 GitHub contribution heatmap 的形式展示我的持续构建轨迹。

它不是日记，也不是生活记录，只展示能证明成长和产出的行动，例如：

- 完成 Brief
- 完成 PRD
- 完成 DESIGN
- 项目开发
- 提交 GitHub
- 页面上线
- 项目迭代
- 复盘更新

每个格子代表一次有效构建行为。点击或悬停时展示简短说明。

这个模块的目标不是让别人读每天发生了什么，而是让人一眼感受到：我在持续推进、持续产出、持续成长。

位置建议：

- 放在成长线路之后，强化“高斜率”
- 或放在项目展示之后，作为持续构建的证据

不要放在首屏，也不要做成独立日记区。

### 4.7 最近更新 / 正在构建

目标：让页面有生命力。

这个模块类似 changelog，但不是技术更新日志，也不是日记，而是成长更新记录。

示例：

```text
最近更新

2026.05
- 完成个人主页 Brief / PRD / DESIGN
- 用 SpecDrivenCoding 搭建第一版个人主页
- 新增成长热力图模块

2026.06
- 上线第一个项目 Demo
- 更新 GitHub 链接
- 补充作品说明
```

这个模块传递的感觉是：

> 这个人不是做了一个网页就结束了，他还在持续迭代。

内容保持短，放 3-5 条即可。

### 4.8 联系与入口

目标：让人能继续了解我。

内容包括：

- 微信或邮箱
- GitHub / 作品入口
- 社交平台
- 可选的一句话邀请

语气要自然，不要营销感太重。

## 5. 视觉设计原则

### 5.1 色彩

整体不能只用单一蓝色、紫色或黑灰。需要有主色、强调色和中性色的区分。

建议方向：

- 背景：干净、稳定
- 主色：体现科技感和行动感
- 强调色：用于成长节点、项目亮点、按钮
- 文字：高对比，保证可读性

颜色要服务信息，不要只是为了炫。

### 5.2 字体

中文必须清晰、现代、有力量。

标题可以更有冲击力，正文要稳定可读。

要求：

- 不使用过小字体
- 不使用难读的花哨字体
- 移动端必须清楚
- 标题和正文层级明确

### 5.3 布局

整体布局要有节奏：

- 首屏强冲击
- 中段高信息密度
- 项目区清晰可扫读
- 结尾轻量收束

桌面端可以更有空间感，移动端要优先保证可读和顺滑浏览。

不要做成一堆普通卡片堆叠，也不要每个模块都像模板区块。

### 5.4 动效

动效只用于强化“成长、路径、推进”的感觉。

可以有：

- 首屏轻微动态
- 滚动时节点出现
- 项目卡片 hover
- 路径线条延展
- 热力图悬停说明

不能有：

- 影响加载的重动画
- 过度花哨的转场
- 让移动端卡顿的效果
- 看起来像炫技但不服务内容的动画

## 6. 交互设计

页面应该是单页式体验，用户通过向下滚动完成理解。

导航可以固定在顶部或轻量悬浮，包括：

- 首页
- 我是谁
- 成长线
- 项目
- 能力
- Build Log
- 联系

交互要求：

- 点击导航能跳转到对应区域
- 手机端导航不能遮挡内容
- 所有按钮都有明确反馈
- 外部链接打开清楚
- 页面加载失败风险要低

## 7. 响应式要求

必须优先保证手机访问体验，因为朋友圈场景大概率来自手机。

手机端要求：

- 首屏 3 秒内看懂主题
- 文字不挤
- 项目卡片纵向排列
- 成长线不能复杂到看不懂
- 按钮和链接容易点击
- 不依赖 hover 才能理解内容

桌面端要求：

- 首屏有视觉冲击
- 内容宽度不要过宽
- 项目区可以多列展示
- 成长线可以更有空间和层次

## 8. 内容语气

语气要自信，但不能油。

可以说：

- 我正在做
- 我尝试过
- 我把想法推进成了作品
- 我在持续迭代
- 这是我的成长路径

避免说：

- 我非常厉害
- 我精通一切
- 我是天才
- 改变世界
- 空泛的大词

核心语气是：有野心，但有证据。

## 9. 必须突出

页面必须突出：

1. 这是一个由我通过 SpecDrivenCoding 做出来的网站。
2. 我有清晰的成长线路。
3. 我做过具体项目。
4. 项目有 Demo 或 GitHub 等证据入口。
5. 我不是只会想，而是会推进。
6. 这个主页未来会继续迭代。

## 10. 必须避免

页面不能出现：

- 日记内容
- 琐碎生活流水账
- 无价值的堆砌
- 打不开或加载很慢的重效果
- 传统简历模板感
- 大段没人愿意读的自我介绍
- 没有证据的自夸
- 手机端体验差

## 11. 开发提示

实现时优先选择稳定、轻量、易部署的方案。

页面应当支持：

- 静态部署
- 快速加载
- 移动端适配
- 后续方便更新内容
- 项目数据可维护
- 不依赖复杂后台

视觉效果要建立在稳定页面之上。任何会明显影响打开速度、可读性或移动端体验的效果都应该放弃。

## 12. 成功标准

设计成功的标准是：

1. 家人看了觉得体面。
2. 老师看了觉得像作品集。
3. 老板看了觉得这个人有潜力。
4. 朋友点进来觉得“你最近变得不一样了”。
5. 我自己愿意把这个网站发出去。
6. 这个页面后续可以继续更新，而不是一次性作品。
