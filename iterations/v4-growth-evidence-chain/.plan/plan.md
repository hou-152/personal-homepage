# v4-growth-evidence-chain · Plan

## 背景

v3 解决了项目证据不足的问题，但页面叙事有“工具和项目抢主角”的风险。本轮要把页面重心调整为“成长证据链”：人是主角，项目是证据，Workbench 是方法。

## 范围

### 做

- 建立 v4 PRD 和 phase 记录。
- 调整首页叙事顺序和关键文案。
- 把项目卡片改成证据卡片。
- 将 Workbench 表达改为作品背后的方法。
- 同步能力地图、成长线、Build Log。
- 做本地构建和桌面 / 移动端视觉检查。

### 不做

- 不新增项目数量。
- 不改技术栈、部署平台或路由结构。
- 不恢复已否定项目。
- 不公开私密路径、账号 ID 或内部原始数据。
- 不替用户 commit / push。

## 阶段总览

| Phase | 目标 | 状态 |
|---|---|---|
| 01-spec-and-narrative | 固化 v4 规格，确认叙事方案 | completed |
| 02-content-data-refactor | 重写页面数据，让项目成为证据链 | completed |
| 03-layout-and-visual-pass | 调整页面结构、移动端样式和视觉层级 | completed |
| 04-verification-and-preview | 构建验证、截图检查、本地预览交付 | pending |

## 关键决策

- 本轮不是增加更多项目，而是改变项目在页面中的叙事角色。
- BRIEF 的方向仍成立，核心是更彻底地执行“个人品牌页为主，作品集作为证据”。
- v3 项目筛选结果继续沿用，不再扩展新项目池。
- Workbench 不能消失，但应从“地图主角”降级为“方法支撑”。
- 每个 phase 完成后停下来让用户验收，再进入下一阶段。
