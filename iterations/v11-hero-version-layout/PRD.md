# v11 Hero Version Layout PRD

## Goal

修正 V10 首屏的信息分配和适配问题：让左栏讲“侯斯博 / Da Capo”这个人，右栏讲《serious AI 内参》这个代表作品，并在顶部明确标注当前版本。

## Problems

- 电脑端首屏高度偏高，第一次打开时右栏内容容易超出第一幕。
- 移动端内容层级偏重，按钮和右栏预览占用过多注意力。
- “把 AI 信息压力做成判断系统”应该属于右栏代表作品，而不是左栏自我介绍。
- Da Capo 的含义没有解释清楚，且“回到开头”容易被误读成倒退。
- 页面已经进入 V10，但顶部没有显示版本状态。

## Scope

- 顶部品牌区显示 `V11` 和 `11 次迭代`。
- 左栏文案顺序改为：姓名 / 作品型定位 / Da Capo 含义。
- Da Capo 解释改为“复杂问题先回到原点，再重新推进”。
- 右栏新增代表作品核心价值句：“我正在把 AI 信息压力，做成可复盘、可交付、可行动的判断系统。”
- 压缩首屏间距、右栏预览高度和移动端布局。
- Build Log 增加 V11 记录。

## Out Of Scope

- 不制作真实视频。
- 不重做 AI 内参完整证据链页面。
- 不更换整体视觉系统。

## Acceptance

- 顶部可见 `V11` 和迭代次数。
- 桌面首屏第一幕能看到左栏主文案、右栏代表作品和主要入口。
- 移动端不横向溢出，阅读顺序为：身份、定位、Da Capo、代表作品。
- `npm run build` 通过。
