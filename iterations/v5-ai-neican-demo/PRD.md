## 1. 项目背景

个人主页的长期目标是向家人、老师、朋友和老板展示高成长、高能量和真实作品证据。v5 迭代新增一个面向招聘方和商业访客的《AI 内参》信息流 Demo，用来证明我能把 AI 信息变成判断，把判断推进成行动。

参见：[BRIEF.md](../../BRIEF.md)

## 2. 本次目标

新增一个专门的 `AI 内参信息流 Demo` 区块，让访问者理解《AI 内参》不是新闻搬运，而是一条真实的信息处理和判断生产流程。

本次迭代要让招聘方看完后形成一个明确判断：这个人能用 Agent 和人工判断搭建一条从 AI 信息输入到团队行动的工作链路。

## 3. 用户与关键场景

### 场景 1：老板或招聘方快速判断我的价值

访问者从项目卡片进入 Demo，想知道“AI 内参编辑流水线”到底证明了什么能力。看完后应理解：我能处理 AI 信息洪水，并把信息转成组织可用的判断材料。

### 场景 2：老师或朋友理解项目背后的真实流程

访问者不关心工具名，但想知道这套系统如何运转。Demo 应该用可读的六步流程解释：信息接入、人工筛选、日报生成、评论入库、深度拆解、判断行动。

### 场景 3：懂 AI 工具的人验证我不是只做包装

访问者知道 Agent 工作流，需要看到流程是真实的、有人机分工、有准入门槛、有深度加工，而不是简单自动摘要。

## 4. 页面结构与信息架构

本次不新增独立页面，只在首页项目区之后、方法区之前新增一个锚点区块：`#ai-neican-demo`。

### AI 内参信息流 Demo 区

- **用途**：展示《AI 内参》的真实生产链路，并把它翻译成招聘方能理解的能力证明。
- **主标题**：把 AI 信息变成判断，把判断推进成行动。
- **副标题**：说明这不是新闻搬运，而是一条真实的信息流：信息进入、人工筛选、Agent 生成日报、评论入库、深度拆解，最后变成团队能使用的判断材料。
- **交互结构**：
  - 播放 / 暂停按钮。
  - 六个步骤按钮，支持手动切换。
  - 当前步骤说明。
  - Agent 负责什么。
  - 我负责什么。
  - 本环节输出什么。
- **六步流程**：
  1. 信息接入：RSS 和公开信息源进入阅读池。
  2. 人工筛选：我判断哪些内容值得进入当日处理。
  3. 生成日报：Agent 整理成可评论、可复盘的日报。
  4. 评论入库：我写下入库理由，形成正式编辑清单。
  5. 深度拆解：Agent 拆文章结构、提炼核心概念、梳理概念关系。
  6. 判断行动：最终形成 AI 内参、团队提醒、选题、Brief 或下一步行动。

## 6. 验收标准

- 桌面端能清楚看到六步流程和人机分工。
- 移动端不溢出、不遮挡，步骤可点击。
- 播放按钮能自动推进步骤，最后停在第六步。
- 项目区 `AI 内参编辑流水线` 的 Demo 按钮跳转到 `#ai-neican-demo`。
- 页面文案不出现小圈子术语作为主叙事，不出现微信、思想孵化、MCP、relation 等干扰内容。
- `npm run build` 通过。
