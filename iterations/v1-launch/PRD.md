# PRD：个人主页

## 1. 项目背景

这是一个个人展示窗口。

它不是普通简历，也不是日记站，而是一个可以被反复转发、反复打开、反复迭代的个人主页。它要让第一次重新认识我的人，在很短时间里感受到：这个人有能量、有成长速度、有作品意识，也有继续变强的潜力。

这个网站本身也是作品的一部分。它要明确传达：这是我用 SpecDrivenCoding 从 Brief、PRD、DESIGN 到开发实现一步步做出来的成果。

## 2. 本次目标

本次目标是做出一个足够酷、足够稳定、足够能展示我的个人主页 MVP。

核心效果：

1. 让家人、老师、朋友、老板看到后觉得：“这个做得不错。”
2. 让老师能把它理解成我的作品集。
3. 让老板能把它理解成我的个人价值展示页。
4. 让朋友圈访问者能快速重新认识我。
5. 让网站本身成为我能力、审美和执行力的证明。

一句话目标：

用一个很酷、很稳、很有成长线索的网站，展示我通过 SpecDrivenCoding 做出来的个人主页，以及我这个人的上升斜率。

## 3. 用户与关键场景

### 3.1 家人

家人不一定理解具体项目，但他们能通过页面感受到我在认真做事、持续成长、有明确方向。

他们需要看到：

- 我在做什么
- 我做出了什么
- 我不是随便玩玩
- 这个东西看起来体面、有成果感

### 3.2 老师

老师会把这个网站当成作品集入口。

他们需要看到：

- 我有哪些项目和尝试
- 我的学习路径和能力变化
- 我能不能把想法做成可展示的东西
- 我是否具备持续产出的能力

### 3.3 朋友 / 朋友圈访客

他们可能只是点进来看一下，停留时间不长。

他们需要在前几屏快速感受到：

- 这个页面有趣
- 这个人最近变化挺大
- 这个人做的东西比想象中认真
- 值得继续往下看

### 3.4 老板 / 潜在合作方

老板会从“这个人有没有价值”角度看。

他们需要看到：

- 我能做什么
- 我做过什么
- 我的成长速度怎么样
- 我的表达、审美、执行力是否靠谱
- 是否值得给我机会、合作、信任或进一步了解

## 4. 产品定位

这个网站不是单纯作品集，而是个人品牌页为主、作品集作为证据。

作品集回答：我做过什么。

个人品牌页回答：你应该如何理解我这个人。

本网站的核心定位是：

我是一个高能量、高成长、能把想法推进成作品的人。项目、GitHub、成长热力图和 Build Log 都是这个判断的证据。

## 5. 页面结构

### 5.1 首屏：首页冲击区

目标：第一眼抓住人，让访问者知道这不是普通个人主页。

内容建议：

- 一句强身份表达
- 一句解释我正在做什么
- 一个明确的视觉主元素
- 体现 SpecDrivenCoding / 自己做出来的网站属性

示例文案：

> 我正在用 SpecDrivenCoding，把自己的成长轨迹做成一个可以被看见的网站。

首屏不应该像普通简历，也不应该堆太多文字。它要先让人觉得“有点酷”。

### 5.2 我是谁

目标：快速建立理解。

内容包括：

- 我的身份
- 我的兴趣方向
- 我正在探索的领域
- 我希望别人如何理解我

这部分不是自我介绍作文，而是个人定位。

### 5.3 成长线路

目标：展示高斜率。

这是整个网站最核心的部分之一。

形式可以是时间线、成长地图、项目阶梯。重点不是罗列经历，而是让人看到：

- 我从哪里开始
- 我做过哪些尝试
- 我遇到过哪些转折
- 我现在在往哪里走
- 我的成长速度为什么值得被注意

### 5.4 项目展示

目标：给出证据。

项目展示不是只放链接，而是每个项目做成一个项目卡片。

每个项目卡片包括：

- 项目名称
- 一句话说明
- 我负责或完成的部分
- 它证明了什么能力
- Demo 链接
- GitHub 链接

GitHub 链接是证据入口，不是页面主角。每个 GitHub 链接旁边最好说明这个仓库证明了什么。

项目选择原则：

1. 最能体现个人价值的项目放前面。
2. 最能体现 SpecDrivenCoding 流程的项目放前面。
3. 太琐碎、太弱、无法解释价值的项目不放。

### 5.5 能力地图

目标：让老师和老板更容易判断我。

不建议使用饼状图，也不建议写“HTML 80%、CSS 70%”这种虚假的技能比例。

能力应该按结果导向表达：

- 把想法变成文档
- 把文档变成页面
- 用 AI 协作推进项目
- 梳理信息结构
- 设计展示路径
- 持续迭代作品
- 公开展示和表达

这一屏体现的是“我能把能力组织成结果”，而不是单纯罗列工具。

### 5.6 成长热力图 / Build Log

目标：证明我在持续推进，而不是偶尔做一次。

用类似 GitHub contribution heatmap 的形式展示我的持续构建轨迹。

它不是日记，也不是生活记录，只展示能证明成长和产出的行动，例如：

- 完成 Brief
- 完成 PRD
- 完成 DESIGN
- 提交 GitHub
- 上线项目
- 更新作品
- 复盘和迭代

每个格子代表一次有效构建行为。点击或悬停时展示简短说明。这个模块的目标是让访问者看到：我不是偶尔做一次，而是在持续推进。

### 5.7 最近更新 / 正在构建

目标：让页面有生命力。

这个模块类似 changelog，但不是技术日志，也不是日记，而是成长更新记录。

示例：

- 2026.05：完成个人主页 Brief / PRD / DESIGN
- 2026.05：用 SpecDrivenCoding 搭建第一版个人主页
- 2026.06：上线第一个项目 Demo
- 2026.06：补充 GitHub 链接和项目说明

它的作用是让别人知道：这个网站不是做完就结束，而是会持续迭代。

### 5.8 联系与入口

目标：让人能进一步了解我。

可以包括：

- 微信 / 邮箱
- GitHub
- 作品链接
- 其他公开展示入口

## 6. 信息架构

网站的信息优先级如下：

1. 第一眼的酷感和完成度
2. 我是谁
3. 我的成长线路
4. 我做过什么
5. 我能提供什么价值
6. 我是否持续推进
7. 如何继续了解我

内容筛选标准：

必须放：

- 能证明成长速度的内容
- 能证明执行力的内容
- 能证明审美和表达的内容
- 能证明我真的做过东西的内容

不要放：

- 日记
- 琐碎生活记录
- 没有证明力的小事
- 过度自嗨但别人看不懂的内容
- 不稳定、打不开、加载慢的复杂功能

## 7. 目标衡量

这个网站成功的标准不是访问量，而是观看后的感受。

定性指标：

- 家人觉得体面
- 老师觉得像作品集
- 老板觉得这个人有潜力
- 朋友觉得这个人最近很不一样
- 我自己愿意反复发、反复改、反复展示

体验指标：

- 页面加载速度足够快
- 手机和电脑都能正常打开
- 首屏 3 秒内能理解这是个人主页
- 主要内容 1 分钟内能看完
- 项目内容可以持续新增

## 8. 范围

### 8.1 本期要做

- 一个完整可访问的个人主页
- 移动端和桌面端都能看
- 有清晰的视觉风格
- 有个人介绍、成长线路、项目展示、能力地图、成长热力图、联系方式
- 能体现这是我用 SpecDrivenCoding 做出来的网站
- 内容结构方便后续更新

### 8.2 本期不做

- 不做博客系统
- 不做日记系统
- 不做复杂后台
- 不做登录注册
- 不做需要频繁维护的动态功能
- 不做过度炫技导致打不开的效果

## 9. 核心原则

1. 高能量：页面必须展示我积极、上升、持续尝试的一面。
2. 高斜率：重点不是我现在有多完美，而是别人能看出我成长很快。
3. 有证据：不能只说自己厉害，要通过项目、作品、路径、GitHub 和 Build Log 来证明。
4. 稳定：网站必须好打开、加载快、移动端可用。
5. 可迭代：它不是一次性作品，而是一个长期展示窗口。
