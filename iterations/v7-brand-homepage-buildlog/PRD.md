# 个人主页 · PRD · v7-brand-homepage-buildlog

> 文件位置：`iterations/v7-brand-homepage-buildlog/PRD.md`

## 背景

当前首页已经有《AI 内参》Demo、完整证据页和项目卡片，但首页第一眼仍然更像“认真整理过的自我说明页”，不够像一个让访客立刻想点进去看的作品现场。

用户已确认本轮方向：

- 公开主名使用 `侯斯博 / Da Capo`。
- 左上角 `SpecDriven` 过于突兀，应从品牌入口退到方法或构建语境里。
- 首页暂时先把《AI 内参》作为最值得看的重点作品。
- Build Log 参考 BrainLog / GitHub contribution 的展示逻辑，但不照搬黑色视觉。
- 本轮完成后直接 commit 并 push 到 GitHub Pages。

## 目标

让首页第一眼从“我在介绍我自己”变成“这里有一个正在被做出来的重点作品”，同时保留个人身份和长期构建证据。

## 本轮范围

### 1. 品牌入口

左上角改为：

```text
侯斯博 / Da Capo
```

`SpecDriven` 不再作为门牌出现。它可以保留在方法、构建或文案语境中，表达“我如何推进作品”。

### 2. 首页 Hero

首页 Hero 改为围绕当前重点作品展开：

```text
侯斯博 / Da Capo

现在最值得看的，是我正在做的《AI 内参》：一个把 RSS、阅读库、日报、笔记和 Agent 串起来的信息流 Demo。
```

按钮优先指向：

- `看 AI 内参 Demo`
- `看完整证据链`

### 3. 我是谁

不做完整简历式自我介绍。只补充必要身份：

- 侯斯博 / Da Capo 是同一个人。
- Da Capo 是用户长期使用的微信网名，含义和“回到起点、重新理解、继续推进”有关。
- 当前最值得看的作品是《AI 内参》。

### 4. Build Log

Build Log 改成“构建节奏”表达：

- 上方：GitHub-style contribution rhythm，用格子展示近期有效构建动作。
- 下方：保留最近构建动作列表。
- 文案强调“不是日记，而是推进证据”。

### 5. 发布

完成本轮后：

- 本地构建通过。
- 更新本地预览。
- commit。
- push 到 GitHub Pages 对应仓库。

## 非目标

- 不重做整站视觉系统。
- 不新增外部依赖。
- 不把《AI 内参》包装成完整产品官网。
- 不替换真实截图，本轮只调整入口和结构。

## 验收标准

- 左上角不再显示 `SpecDriven`。
- 首页首屏明确显示 `侯斯博 / Da Capo`。
- 首页首屏直接引导到《AI 内参》Demo 和完整证据链。
- Build Log 看起来像“持续构建节奏”，不再只是少量编号方块。
- 桌面和移动端没有明显文字溢出。
- `npm run build` 通过。
- 代码已 commit 并 push。
